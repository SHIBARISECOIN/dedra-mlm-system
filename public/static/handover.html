정산 흐름

관리자 "일일 ROI 정산" 버튼 클릭
  └→ api.js: runDailyROISettlement(adminId, targetDateStr)
       ├→ [중복 방지] settlements 컬렉션에서 당일 정산 여부 확인
       ├→ [데이터 로드] 모든 active 상태의 investments 조회
       ├→ [각 투자별 처리]
       │    ├→ 만기 체크 (만기 시 status = 'completed')
       │    ├→ 지급 패턴 체크 (daily / weekday / custom)
       │    ├→ D = amount × (roiPercent / 100)  ← 일일 ROI 금액
       │    ├→ wallet.bonusBalance += D  (ROI 지급)
       │    ├→ bonuses 컬렉션: { type:'roi_income', amount:D }
       │    │
       │    ├→ _processUnilevelBonus(D)
       │    │    └→ 상위 추천인 최대 10단계 순회
       │    │       G0 제외, 각 레벨별 요율 × D 지급
       │    │
       │    ├→ _processDirectBonus2Gen(D)  ← 설정 enableDirectBonus2Gen=true 시
       │    │    └→ 1대: directBonus2GenRate1%, 2대: directBonus2GenRate2%
       │    │
       │    ├→ _processCenterFee(D)  ← 설정 enableCenterFee=true 시
       │    │    └→ 해당 센터장에게 centerFeeRate% 지급
       │    │
       │    └→ _processRankGapBonus(D)  ← 설정 enableRankGapBonus=true 시
       │         └→ 직급차 × ratePerStep% 풀 분배
       │
       └→ settlements 컬렉션에 정산 기록 저장

절대 하지 말 것: Firestore 복합 쿼리
where + orderBy 조합은 복합 인덱스가 없으면 에러 발생!
반드시 단일 where만 사용하고 정렬은 JS에서 처리하세요.
// ❌ 금지 - Firestore 인덱스 에러 발생
query(collection(db,'bonuses'), where('userId','==',uid), orderBy('createdAt','desc'))

// ✅ 올바른 방법 - 단일 where + JS 정렬
const snap = await getDocs(query(collection(db,'bonuses'), where('userId','==',uid)));
const sorted = snap.docs.map(d => d.data())
  .sort((a,b) => (b.createdAt?.seconds||0) - (a.createdAt?.seconds||0));
